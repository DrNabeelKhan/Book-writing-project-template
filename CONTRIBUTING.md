# Contributing Guide

## Branching Model

- `main` — stable, buildable manuscript
- `develop` — integration branch
- `feature/<chapter-or-topic>` — individual work branches

Examples:

- feature/ch01-first-draft
- feature/ch02-diagrams
- feature/global-style-pass

## Commit Message Style

Use prefixes:

- ch01:
- appendix-a:
- diagrams:
- build:
- repo:

## Pull Requests

Every PR must include:

- Summary of changes
- Motivation
- Files changed
- Reviewer notes

CI must pass before merging.
